When Why & How?

While CSS Grid is more useful for two-dimensional layouts, CSS Flexbox provides straightforward code for flexible one-dimensional layouts. For example, this page is structured with a vertical navbar down the left of the page as one flexbox container, one vertical main flexbox container, and one horizontal flexbox footer across the bottom. CSS Grid is used to help position these three one-dimensional flexboxes on a two-dimensional page.

  • display: flex; turns a container into a flexbox
  • flex-direction defines whether the container will run as a horizontal row or a vertical column

Example: The green navbar flexbox and the gray main flexbox are both set to column, while the dark gray footer is set to a row flex-direction. Furthermore, each main section within the gray main container is a column flexbox itself.

Positioning Items

When a container becomes a flexbox, its direct children (and only its direct children) become flexible items that can be controlled by flex properties defined in the parent CSS. In order for the header, paragraph, and list elements to become flex items, each main section is also given the display: flex; property.

justify_content and align_items are the two main properties that are applied to the flexbox to position the items within it.

  • justify_content aligns items along the main axis (horizontal for rows, vertical for columns) and takes the following values
    • center which aligns items at the center of the container along main axis
    • flex-start aligns items at the start of the main axis
    • flex-end aligns at the end of main axis
    • space-around aligns items along main axis with spacing before, between, and after
    • space-between aligns items along main axis with spacing between
  • align_items is similar, but aligns items along the cross axis (vertical for rows, horizontal for columns) and takes the following values
    • center
    • flex-start
    • flex-end
    • stretch items stretch to fill container (default)
    • baseline aligns items to text baseline, that the letters "sit" on

Example: The green navbar, a flexbox with a column flex-direction has a justify-content value of space-around to provide the spacing down along the vertical main axis. The dark gray footer has a value of baseline for its align-items property in order to have all the items sitting on the same line, regardless of font-size.

Handling Box Overflow

By default, Flexbox only accommodates one row or column along the main axis. By changing the value of flex-wrap from the default of no-wrap to wrap we can push items that don't fit well down to a second row from left-to-right or over to a second column from top-to-bottom.

wrap-reverse will do the same, except order items from right-to-left for rows and bottom-to-top for columns.

Item Properties

Properties can applied to individual items that are children of flexboxes.

  • flex-shrink will shrink the item if the width the flex container is smaller than the combined widths of the items within it. A numerical value is given to determine for how much an item should be shrunk in comparison to other items sharing its container
  • flex-grow operates similarly, except the objects will grow as space within the flex container allows
  • flex-basis sets the starting size using standard size properties before the above adjustments are made.
  • flex combines all of the above three into a shorthand flex: 1 0 5em; with the first value corresponding to grow, the second to shrink, and the third to basis.
  • order can be used on items with a numerical value to change the order they appear in the flex container
  • align-self can be directly applied to items to override the alignment set in its parent element and takes the same values as the align-items property

Example: The p element in the footer has the property flex: 3 1 150px; while the a elements all have flex: 1 1 150px;. This means that while they all have a starting width of 150px, the element with the text "Resources:" will grow 3 times as much as the links, and they will all shrink at the same rate. So, when the footer is 600px wide, the space for all four elements is divided equally. As the footer grows, 3 times as much of the extra space is allocated to the p element as to each a element.

Responsive Use

Flexbox can be even more useful when used along with media queries for styling responsive pages.

Example: This page has a media query set for max-width of 650px. Under this query, in addition to the grid structure changing to one column, the footer flex direction is changed from row to column, and the flex values of its items are changed to flex: 1 1 0;. This allows for most of the spacing to be adjusted simply by applying justify-content: space-around; and align-items: center: to the footer flexbox.
